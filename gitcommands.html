Below are some of the common Git Commands:


  1. git init

    2. While in the directory you wish to make a git repository the above command will initialize a git repository in that directory.

  2. git add 'file or directory name'

    3. Adds files in the to the staging area for Git. Before a file is available to commit to a repository, the file needs to be added to the Git index (staging area). There are a few different ways to use git add, by adding entire directories, specific files, or all unstaged files.

  3. $ git commit -m "comit message in quotes"

    4. Record the changes made to the files to a local repository. For easy reference, each commit has a unique ID.

  4. $ $ git push "remote_URL/remote_name" "branch"

    5. Sends local commits to the remote repository. git push requires two parameters: the remote repository and the branch that the push is for.

  5. $ $ git pull "branch_name" "remote_URL/remote_name"

    6. Is used to get the latest version of a repository.

  6. $ $ git merge "branch_name"

    7. integrates branches together.

  7. $ $ git checkout -b "new_branch" $ $ git checkout "branch_name"

    8. Used to start working on a different branch. The -b option creates a new branch.
